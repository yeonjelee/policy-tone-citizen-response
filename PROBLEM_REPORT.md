# 데이터 수집 문제 진단 보고서

작성일: 2026-06-16 · 점검 대상: `raw/`, `scripts/`, 디버그 HTML

수집된 데이터와 수집 스크립트를 대조해 **6개 문제**를 확인했다. 심각도 순으로 정리한다.

---

## 1. (치명) 한국은행 결정문·의사록 0건 — 원인 확정 (디버그 HTML 분석)

`raw/bok_mpb/`, `raw/bok_minutes/` 둘 다 **빈 폴더**다. 이 프로젝트의 메인 변수(`tone_inflation`, `tone_growth`, `tone_finstab`)는 한은 결정문·의사록 어조에서 나오는데, 원문이 한 건도 없다.

`--debug`로 저장된 HTML(`debug_bok_mpb_*.html`)을 분석해 **진짜 원인 2가지**를 확정했다.

**(1) 연도 선택이 GET이 아니라 POST다.** 목록 페이지는 `?year=YYYY`를 무시하고, 폼 `stsfdgForm`(method=post)의 `<select name="pYear">`로 연도를 바꾼다. 그래서 `year=2010`이든 `2024`든 **항상 기본(최근) 연도 페이지 1장만** 돌아왔다. 실제로 저장된 18개 디버그 파일이 **모두 정확히 같은 477,908바이트**였던 것이 그 증거다.

**(2) 결정문·의사록 본문이 HTML이 아니라 HWP/PDF 첨부파일이다.** 표 구조는
`회의일자(th) | 결정문(td0) | 기자간담회(td1) | 의사록(td2) | 이슈(td3)` 이고, 각 셀의 링크는
`/portal/cmmn/file/fileDown.do?...&fileSn=N` 형태의 HWP·PDF 다운로드다. 이전 크롤러는 HTML 본문 셀렉터로 텍스트를 긁으려 했으니 애초에 대상이 없었다. 또 회의일자 셀에 **연도가 없어**(`01월 15일(목)`) 날짜 파싱도 실패했다.

→ 수정(v7): `bok_common.py`에서 **POST(pYear)**로 연도별 표를 받고, 결정문=td0 / 의사록=td2 의 **PDF를 내려받아 pdfplumber로 텍스트 추출**. 연도는 요청한 pYear로 부여. 저장된 디버그 HTML로 파서를 검증해 결정문·의사록 PDF 링크가 정확히 추출됨을 확인했다. (POST 응답·PDF 다운로드 자체는 외부망 차단 환경이라 사용자 PC 실행으로 검증 필요.)

## 2. (치명) 실업률 데이터가 실업률이 아님 — 잘못된 지표 수집

`raw/kosis/unemployment_quarterly.csv` 값이 **8,243 ~ 9,060** 범위다. 한국 실업률은 보통 2~4%이므로 이 숫자는 실업률(%)이 아니라 **취업자 수(천 명)** 등 다른 항목이다.

추가로 시계열에 **인공적 단절**이 보인다.

| 분기 | 값 | 비고 |
|---|---|---|
| 2017Q4 | 9,060 | |
| 2018Q1 | 8,442 | 약 -600 급락 |
| 2022Q4 | 8,685 | |
| 2023Q1 | 8,243 | 약 -440 급락 |

이런 점프는 `fetch_kosis()`가 `objL1="ALL"`로 **모든 분류값(연령·성별·전체 등)을 한꺼번에 받아 단순 평균**하기 때문이다. 표 구성이 시점마다 바뀌면 평균 대상 집합이 달라져 값이 튄다. 게다가 현재 스크립트의 `is_valid_rate()`(최댓값<30 검증)를 통과할 수 없는 값이 저장돼 있다 → **현재 디스크 데이터는 검증 로직이 없던 옛 버전 산출물**이다.

→ 수정안: 실업률(%) 단일 항목만 선택(총계 1개 분류), 0~30% 검증, ECOS 1순위. `scripts/collect_kosis.py` 참조.

## 3. (심각) 원/달러 환율이 2011Q4에서 절단 — 1만 행 응답 제한 초과

`raw/ecos/exchange_rate.csv`에 **8개 분기(2010Q1~2011Q4)**만 있다. 60분기 중 52분기 결측.

원인: `collect_ecos.py`가 `731Y001`(환율, **일별**)을 `/1/10000/`(1~10000행)으로 요청한다. 731Y001에는 USD뿐 아니라 수십 개 통화가 들어있어, 일별 × 다통화면 **1만 행이 2010~2011년 구간에서 소진**된다. 필터(`달러`)는 응답을 받은 뒤에 적용되므로, 애초에 USD의 2012년 이후 행이 응답에 포함되지 않는다.

→ 수정안: 항목코드(USD)를 **요청 URL 경로에 직접 지정**해 USD 행만 받도록 변경 + 월 평균 집계. `scripts/collect_ecos.py` 참조.

## 4. (심각) 기재부 키워드 필터 오매칭 — '추경호'(차관 이름)가 '추경'에 걸림

`raw/moef/`에 `추경호 차관 ... 방문/면담/축사/출국` 같은 **정책과 무관한 동정 보도 88건**이 들어와 있다. 원인은 부분 문자열 매칭:

```python
POLICY_KEYWORDS = [..., "추경", ...]
any(kw in title for kw in POLICY_KEYWORDS)   # "추경" in "추경호 차관 방문" == True
```

당시 기재부 차관 이름이 **추경호**라서, 이름이 포함된 모든 보도자료가 정책 문서로 오인 수집됐다.

추가로 `월간 재정동향` 발간 안내가 **133건** 섞여 있다(단순 통계 배포, 재정 어조 신호 없음).

→ 수정(사용자 확정): 노이즈 + 월간동향 모두 제외. 정확 구문/제외어 기반 필터로 교체. `clean_moef.py` 미리보기 검증 결과 **498건 → 실질 정책문서 90건 보존, 추경호 0건**. `scripts/collect_moef.py`, `scripts/clean_moef.py` 참조.

## 5. (심각) 기재부 본문에 페이지 보일러플레이트 혼입

수집된 moef 본문 앞부분이 `바로가기 메뉴 / 본문 바로가기 / 이 누리집은 대한민국 공식 전자정부 누리집입니다. ...` 로 시작한다. `extract_article_text()`의 본문 셀렉터가 실제 구조와 안 맞아 **`body` 전체 fallback**으로 떨어졌고, GNB/푸터/안내 문구가 본문에 들어갔다.

→ 수정안: 본문 컨테이너 셀렉터 보강 + 보일러플레이트 라인 정규식 제거. `clean_moef.py --apply`가 기존 파일 본문도 재다운로드 없이 정제. `scripts/collect_moef.py` 참조.

## 6. (구조적 제약) 네이버 검색 트렌드는 2016Q1부터만 존재

`raw/naver_trends/*`는 **2016Q1~2024Q4(36분기)**만 있다. 네이버 데이터랩이 2016-01부터 제공하기 때문으로, 버그가 아니라 외부 한계다. 반면 정책·거시 변수는 2010Q1~2024Q4다.

**확정 방침: 분석 구간을 2016Q1~2024Q4(36분기)로 축소.** 거시·환율·실업률 재수집도 2016+ 기준으로 맞춘다(기존 60분기 시계열은 분석 시 2016+로 트림).

부가 유의점 — **검색량 정규화 비교 불가**: 네이버 데이터랩은 요청별 최댓값=100 정규화를 한다. `collect_naver.py`는 6개 키워드를 각각 별도 요청으로 받으므로, 변수 간 절대 수준 비교는 무의미하다. 군집 분석에서 각 변수를 표준화(z-score)하면 대부분 상쇄되지만, "어느 키워드가 더 많이 검색됐나" 식 해석은 금물이다.

# 정책 어조에 따른 시민 반응 분석

숙명여자대학교 「데이터분석 및 활용」 개인 프로젝트

> 통화·재정정책의 **어조(tone)** 를 텍스트마이닝으로 수치화하고, 시민의 검색·실현행동·언론보도·금융시장 데이터와 결합하여 **정책–시민 반응 구조**를 실증한 데이터 분석 프로젝트 (2016–2024).

데이터분석 및 활용 · 개인 프로젝트. 최종 산출물은 14페이지 결과 보고서(`정책_시민반응_결과보고서.pptx` / `.pdf`).


---

## 연구 질문

거시경제 이론(비교정태)은 정책이 작동하는 **방향**은 제시하지만, 시민이 어떤 **속도·경로**로 반응하는지는 답하지 않는다.

- **H1 (메인)** — 정책기조(긴축/완화) 변화가 시민 반응(검색)과 이론 정합적으로 공행(共行)하는가?
- **H2 (보조)** — 동일 정책에서도 언론 보도논조에 따라 반응이 조절(moderation)되는가?

## 핵심 발견

- **주의 ≠ 행동의 시간 분리** — 검색(주의)은 정책 직후 수일 내 즉각 반응, 실현행동(주택가격·가계대출)은 6–12개월 시차로 반응.
- **정보 속도의 위계** — 금융시장(선반영) → 시민 주의(사후) → 실현행동(최후행). 시민은 보도를 매개(약 4일 선행)로 반응.
- **6개 잠재국면** — 2단계 중첩 군집으로 정책–시민 관계를 유형화. 이론 이탈 시점(2020-03 코로나, 2024-12 등)은 외생충격으로 해석.
- **외적 타당성** — 네이버·구글·유튜브에서 핵심 결과 재현.

## 분석 기법

어조 분류(텍스트마이닝) · 잠재국면 군집(계층군집 + K-평균, 메인) · 구조적 단절 탐지(이론괴리 z-점수 + LOF) · 편상관/시차 교차상관 · 사건연구(event study) · 플랫폼 교차검증.

## 폴더 구조

```
.
├── scripts/              # 수집 → 전처리 → 분석 → 보고서 생성 코드 (Python + Node)
├── processed/
│   ├── analysis/         # 분석 결과 CSV (군집·이상치·사건연구·편상관 등)
│   ├── events/           # 정책 이벤트 일자
│   ├── figures/          # 시각화 (t-SNE, 히트맵, 타임라인)
│   └── tone/             # 어조 산출물 (※ 대용량 중간파일은 .gitignore)
├── METHODOLOGY.md        # 방법론·파라미터 상세
├── REFERENCES.md         # 참고 문헌
├── PROBLEM_REPORT.md     # 데이터 수집 문제·해결 기록
├── ANOMALY_NOTES.md      # 이상치 해석 노트
├── requirements.txt
├── .env.example          # API 키 템플릿
└── 정책_시민반응_결과보고서.pptx / .pdf   # 최종 보고서 (익명본)
```

> **원본 데이터(`raw/`)는 포함하지 않습니다.** AI Hub·빅카인즈 기사·네이버 데이터랩 등은 용량(5GB+) 및 재배포 라이선스 제약이 있어, `scripts/`로 직접 수집해야 합니다.

## 데이터 출처

| 구분 | 출처 |
|---|---|
| 정책 어조 | 한국은행 통화정책 결정문·금통위 의사록, 기획재정부 재정 발표문 |
| 시민 검색 | 네이버 데이터랩, Google Trends(검색·YouTube) |
| 언론 보도 | 빅카인즈(BigKinds) |
| 거시 통제 | 한국은행 ECOS, 통계청 KOSIS |
| 금융시장 | 원/달러 환율(ECOS), KOSPI(yfinance) |
| 보조 | AI Hub 금융 합성 데이터 |

## 재현 방법

```bash
pip install -r requirements.txt
cp .env.example .env          # ECOS·네이버·KOSIS API 키 입력

# 수집 (raw/ 생성)
python scripts/collect_ecos.py
python scripts/collect_naver.py
# ... (scripts/ 내 collect_*.py)

# 분석 파이프라인
python scripts/score_tone.py
python scripts/cluster_phases.py
python scripts/deviation_anomaly.py
python scripts/event_study.py
# ...

# 결과 보고서 생성
node scripts/build_deck.js
```

## AI 도구 활용

LLM(Claude)은 **보조 도구**로 활용. 데이터 수집·정제 코드 구현, 디버깅, 시각화 등 반복·기술 작업에 사용. 연구설계·방법론·해석 등 학술적 판단은 연구자 본인이 수행. 상세 구분은 보고서 「AI 도구 활용 내역 보고」 참조.
